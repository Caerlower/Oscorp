# x402 (Algorand TestNet)

Oscorp uses the **real x402 v2 HTTP payment protocol** with `@x402-avm` packages and the GoPlausible facilitator.

## Flow

1. Frontend calls a paid API (e.g. `POST /api/agents/reddit`) via `createX402Fetch()`.
2. Backend returns **HTTP 402** with `PAYMENT-REQUIRED` header (base64 JSON).
3. Browser x402 client builds + signs the Algorand USDC payment.
4. Client retries with `PAYMENT-SIGNATURE` header.
5. Backend forwards to `https://facilitator.goplausible.xyz` for **verify** + **settle**.
6. Backend returns agent content with `PAYMENT-RESPONSE` header.

## Treasury wallet

Single source of truth: `shared/payment-constants.json` (`recipientAddress`). Loaded by `backend/app/core/payment_constants.py` and the frontend payment constants module.

## Frontend

- `x402-payer/src/index.ts` — `createX402Fetch()` library
- `frontend/src/hooks/useX402Fetch.ts` — Web3Auth / Pera wallet integration
- Per-action mode: confirmation modal before signing
- Agent wallet mode: auto-approve, signs with derived agent key

## Optional Node proxy (`x402-payer/`, port 8110)

For scripts or server-side paid fetch:

```bash
cd Oscorp/x402-payer
cp .env.example .env
npm install
npm run dev
```

`POST /fetch` with `{ url, method?, json?, payerMnemonic? }`.

## Backend env

```bash
FACILITATOR_URL=https://facilitator.goplausible.xyz  # also hardcoded in middleware
```

## Test

```bash
curl -i -X POST http://127.0.0.1:8000/api/agents/reddit \
  -H "Content-Type: application/json" \
  -d '{"productInfo":"test"}'
```

Expect `402` with `PAYMENT-REQUIRED` and `X-PAYMENT-REQUIREMENTS` headers.
